# Security Policy

## Our Commitment to Safety
Kinesis.rs is built for automated trading, where security is the absolute priority. We employ a "Zero-Trust" architecture to ensure private keys are never exposed or uploaded.

## Reporting a Vulnerability
If you discover a potential security vulnerability in this project, please **do not create a public issue**. Instead, please report it through one of the following channels:

1. **GitHub Private Vulnerability Reporting**: Use the "Security" tab in this repository to "Report a vulnerability".
2. **Email**: (Optional) [Insert your security email here if applicable]

We will acknowledge your report within 48 hours and provide a timeline for resolution.

## Supported Versions
We only provide security updates for the latest stable release.
