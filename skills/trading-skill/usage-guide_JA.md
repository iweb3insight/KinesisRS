# FreedomAgent 詳細使用ガイド

## 1. コア設計理念
FreedomAgent は **エージェントファースト** な取引システムです。すべての設計は、LLM エージェント（Gemini、Claude など）が複雑な取引を安全かつ正確に実行できるように行われています。

- **ステートレス (Stateless)**: 各コマンドには実行に必要なすべてのコンテキストが含まれています。
- **JSON ファースト**: エージェントが `TradeResult` を正確に解析できるように、常に `--json` フラグを使用することをお勧めします。
- **セキュリティ重視**: シミュレーションによる検証を強制するため、デフォルトで `--dry-run` が有効になっています。

## 2. インタラクションのベストプラクティス（エージェント向け）

### 購入ワークフロー
1. **見積もり (Quote)**: `quote` を実行します。`amount_out` を解析してユーザーに提示します。
2. **リスク評価**: `buy --dry-run` を実行します。
   - `stages` 内の `duration_ms` を確認します。
   - `gas_estimate` を確認します。
   - 成功した場合は、シミュレーション結果を表示し、ユーザーに二次確認を求めます。
3. **実際の取引**: ユーザーの確認後、`buy --no-dry-run` を実行します。

### 売却ワークフロー
- 売却コマンドは、承認 (`approve`) が必要かどうかを自動的に検出します（BSC の場合）。
- `stages` に `approve` ステージが表示された場合、承認操作が行われたことを意味します。

## 3. Solana 固有の機能

### Jito Bundle 加速
Solana チェーンでは、フロントランニング（MEV）を防止したり、極端な混雑時に取引を成立させたりするために、Jito を使用する必要があります。
```bash
./kinesis_rs buy <TOKEN> 0.1 --chain solana --jito-tip 0.001
```
- **パラメータ**: `--jito-tip` の単位は SOL です。推奨範囲: 0.0001 - 0.01。

### Raydium スマートルーティング
Pump.fun 以外のトークン（USDC、SOL/USDT プールなど）の場合、エグゼキューターは Raydium V3 Trade API を自動的に呼び出して最適なパスを探索します。

## 4. よくあるエラーコードと対処法

| エラーメッセージ | 原因 | 提案 |
| :--- | :--- | :--- |
| `AccountNotFound` | ウォレット残高が 0 または未初期化 | ネイティブトークン (BNB/SOL) を入金してください |
| `SlippageExceeded` | 価格変動が大きすぎる | `--slippage` を増やしてください (例: 25.0) |
| `RouteNotFound` | 流動性不足または API 未インデックス | トークンアドレスを確認するか、小額で試してください |
| `Simulation failed` | ロジック実行がリバートされた | `raw_revert_data` を確認して詳細を把握してください |

## 5. パフォーマンス監査
`TradeResult` 内の `stages` 配列を解析することで、エージェントは以下を計算できます。
- **API レイテンシ**: `quote` ステージの `duration_ms`。
- **実行レイテンシ**: `buy`/`sell` ステージの `duration_ms`。
- **総所要時間**: すべてのステージの `duration_ms` の合計。
